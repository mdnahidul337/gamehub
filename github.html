Git Workflow Guide

A modern, clean guide to managing your Git projects.

How to Start a New Project

Follow these steps to get your new project folder onto GitHub for the first time.

Step 1: Initialize Git

In your project's root folder, run this command to create a new Git repository.

git init

Step 2: Make Your First Commit

Add all your files and create the first "snapshot" (commit) of your project.

git add .
git commit -m "Initial commit"

Step 3: Link to GitHub

1. Go to GitHub and create a new, empty repository (don't add a README or .gitignore).

2. Copy the URL GitHub gives you.

3. Paste your URL below. The command will update live.

git branch -M main
git remote add origin <your-repo-url>

Step 4: Push to GitHub

Send your "Initial commit" up to GitHub. The -u flag sets it as the default, so next time you can just run git push.

git push -u origin main

Common Problems & Solutions

Here are some common issues you might face when uploading files and how to solve them.

"Failed to push. Remote is ahead." +

Problem: You see an error like Updates were rejected because the remote contains work that you do not have locally.

Solution: This is very common! It just means someone else pushed code to GitHub while you were working. You just need to sync their changes *before* you push yours.

This is exactly what Step 4 in the "Update Project" tab is for!

git pull origin main --rebase
"I made a typo in my last commit message." +

Problem: You just committed, but you spelled something wrong in the message (e.g., "Fiexd bug").

Solution: As long as you have not pushed the commit to GitHub yet, this is an easy fix. Just run this command:

git commit --amend -m "My new correct message"
"I forgot to add a file to my last commit." +

Problem: You committed, but then realized you left out a file that should have been included.

Solution: Again, as long as you have not pushed, this is simple. Just add the file(s) and run the "amend" command. It will add the new files to your *previous* commit.

git add path/to/forgotten-file.js
git commit --amend --no-edit

(The --no-edit flag just re-uses your last commit message automatically.)
